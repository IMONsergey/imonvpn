# 12. План максимальной устойчивости

Этот документ описывает не просто рабочий VPN, а **устойчивую многоустройственную схему** для:

- `Hiddify` / Xray-клиентов
- `AmneziaVPN` / `AmneziaWG`
- нескольких личных устройств
- постепенного усиления защиты от блокировок

## Текущее состояние

На сегодня уже подтверждено:

- `Hiddify` на macOS работает
- прямой `Xray Reality` через наш VPS работает
- сервер переживает reboot и поднимает `xray-reality` автоматически

Это хороший Layer 0, но не финальная форма для "максимальной устойчивости".

## Целевая архитектура

### Layer A — основной канал для Hiddify

- `Xray Reality`
- минимум два живых профиля:
  - primary
  - backup
- отдельные клиентские ключи/UUID **по устройствам**, а не один общий ключ на всё

### Layer B — основной канал для Amnezia

- `AmneziaWG 2.0`
- установка через `AmneziaVPN` по SSH
- для каждого устройства — отдельный guest key / config

### Layer C — маскировка через домен

- собственный домен
- реальный `A`-запись на VPS
- обычный сайт-заглушка на домене
- домен используется как дополнительный легитимный внешний слой для сервера

### Layer D — резерв по инфраструктуре

- второй VPS
- другой провайдер
- желательно другой ASN / другая сеть / другая страна в Европе
- назначение:
  - failover
  - relay
  - запасной внешний IP, если первый IP начинают душить

## Почему именно так

Один IP + один транспорт + один протокол — это удобно, но хрупко.

Более устойчивая схема строится на диверсификации:

- разные протоколы: `Reality` и `AmneziaWG 2.0`
- разные клиентские экосистемы: `Hiddify` и `AmneziaVPN`
- разные точки отказа: домен, прямой IP, второй VPS
- отдельные ключи для устройств: проще отзывать доступ и отслеживать проблемы

Это частично вывод из официальной документации и частично инженерная **инференция**:
- официально `AmneziaWG 2.0` позиционируется как более устойчивый к DPI
- официально `Hiddify` поддерживает `Reality`, single-config import и subscription links
- далее я делаю вывод, что комбинация нескольких независимых путей устойчивее одного

## Что именно внедряем

### 1. Для Hiddify

- оставить текущий рабочий `Reality`
- не использовать один общий профиль на все устройства
- выдать отдельный профиль на:
  - `Mac`
  - `Android TV`
  - `Android phone` / `tablet` при необходимости
- подготовить нормальную subscription URL

### 2. Для Amnezia

- подключить сервер в `AmneziaVPN`
- установить `AmneziaWG 2.0`
- выпускать guest access отдельно на каждое устройство

Важно:

- по официальной документации Amnezia guest access работает только для **одного устройства** на один guest key
- full access key можно использовать на любом числе устройств, но он даёт полный контроль над сервером и не должен раздаваться

### 3. Для домена

- подключить домен к текущему VPS
- поднять обычную web-заглушку
- позже добавить `Cloudflare`-слой как отдельный backup-сценарий, а не как единственную опору

### 4. Для управления доступами

- один профиль/ключ = одно устройство
- вести таблицу устройств
- уметь быстро:
  - отозвать один ключ
  - выдать новый
  - не трогать остальные устройства

### 5. Для сервера

- убрать SSH по паролю
- перейти на SSH-ключи
- сменить root password
- проверить firewall и сервисы после перехода

## Практический план по этапам

### Этап 1 — довести текущий VPS до production-вида

- привести SSH в порядок
- убрать парольный доступ
- сделать отдельные Hiddify-профили по устройствам
- сохранить и документировать живые ссылки

### Этап 2 — добавить AmneziaWG 2.0

- установить `AmneziaWG 2.0` через AmneziaVPN
- сделать guest configs для:
  - iPhone
  - Android
  - Android TV при необходимости

### Этап 3 — подключить домен

- завести домен на VPS
- повесить реальную заглушку-сайт
- подготовить отдельный резервный сценарий вокруг домена

### Этап 4 — второй VPS

- взять второй дешёвый VPS у другого провайдера
- использовать как резерв или relay
- не размещать оба VPS у одного и того же провайдера

## Что нужно от пользователя для максимальной устойчивости

Без этого дальше будет только "рабочий VPN", но не "максимально устойчивый":

1. Домен
2. Второй VPS у другого провайдера
3. Установленный `AmneziaVPN` хотя бы на одном устройстве, где можно сделать full-access подключение к серверу

## Что уже можно считать правильным решением

Если цель — не просто "работает на одном ноутбуке", а "живёт на нескольких устройствах и переживает часть блокировок", то базовая правильная форма проекта такая:

- `Reality` для `Hiddify`
- `AmneziaWG 2.0` для `Amnezia`
- домен как дополнительный внешний слой
- второй VPS как инфраструктурный резерв
- отдельные ключи по устройствам

Это и есть направление, в котором дальше ведём проект.
