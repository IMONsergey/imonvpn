# План под AmneziaVPN

## Почему берём Amnezia как основной клиент

По официальной документации AmneziaVPN поддерживает **XRay VLESS Reality** и умеет работать с self-hosted серверами:

- список поддерживаемых протоколов: [Installing and Configuring Protocols](https://docs.amnezia.org/documentation/instructions/install-configure-protocols/)
- описание XRay / Reality: [XRay](https://docs.amnezia.org/documentation/xray/)
- поддерживаемые форматы конфигурации: [Supported Configuration Formats](https://docs.amnezia.org/documentation/supported-configuration-formats/)

Из этих источников следует:

1. AmneziaVPN поддерживает **AmneziaWG** и **XRay VLESS Reality**.
2. XRay Reality доступен на всех основных платформах Amnezia.
3. Amnezia умеет импортировать XRay-конфигурации, но не обязательно создаёт все варианты VLESS-схем автоматически из любой внешней конфигурации.

## Какой путь выбираем

Для первой итерации берём самый прагматичный маршрут:

1. Используем **нативный self-hosted flow Amnezia** для первой установки сервера через SSH.
2. Сначала ставим **AmneziaWG 2.0**.
3. Затем добавляем **XRay VLESS Reality** как резервный протокол.
4. Используем текущие серверные скрипты этого репозитория в первую очередь как:
   - техническую базу
   - источник hardening-практик
   - резервный вариант ручной настройки XRay
5. Только потом решаем, нужно ли:
   - дописывать собственные скрипты под Amnezia-native сценарий
   - добавлять relay / Cloudflare / WebRTC
   - перерабатывать исходные XRay-скрипты глубже

## Почему план изменился

После уточнения пользовательского сценария стало ясно, что VPN должен в первую очередь быть устойчивым **из России**, включая уже встречавшиеся блокировки. Поэтому теперь логика такая:

- **AmneziaWG 2.0** официальный стек Amnezia продвигает как более трудный для DPI-детекта за счёт маскировки под обычные UDP-протоколы
- **XRay Reality** у них же остаётся поддерживаемым и полезным резервом
- для реальной эксплуатации в России безопаснее иметь **минимум два разных протокольных пути**

## Наш первый технический milestone

Нужно получить рабочую схему:

`AmneziaVPN client -> AmneziaWG 2.0 on EU VPS`

и параллельно:

`AmneziaVPN client -> XRay VLESS Reality on the same EU VPS`

Критерий готовности:

- клиент подключается из Amnezia на `Mac`, `iPhone`, `Android`
- основной протокол работает из российских сетей
- резервный протокол готов к переключению
- зарубежные сервисы открываются через VPN
- российские сервисы при необходимости работают через split tunneling
- нет явных DNS / QUIC / WebRTC утечек на приоритетных устройствах
